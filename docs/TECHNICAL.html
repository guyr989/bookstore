Bookstore — Technical Reference

Built incrementally during a guided review, one core method paired with its covering test(s) at a time. Only methods whose covering test scored 8+ get a row — weaker tests are dropped from the suite entirely rather than documented. The "Covering Test(s)" column names the test(s) that earned the method its row.

GetByIsbn in: string isbn → out: Result<Book> Finds one book by ISBN; joins its <author> elements into Authors and the comma-separated AuthorsDisplay string. Returns a failed Result (NotFound) instead of null, so reads and writes share one failure convention. Add in: Book book → out: Result Validates the book; if valid and the ISBN is not already taken, appends it as a new <book> element and saves — serializing every field, including the optional cover attribute and multiple <author> elements. Returns a failed Result (ValidationFailed or Conflict) instead of throwing for these expected, non-exceptional outcomes. Edit in: Book book → out: Result Validates the book; if valid and a book with the same ISBN exists, replaces it entirely (all fields) and saves. Returns a failed Result (ValidationFailed or NotFound) instead of throwing for these expected, non-exceptional outcomes. BookValidator (field validation); FileVersionStore (optional, snapshots the new state) XmlSchemaValidationException if the underlying file fails schema validation Edit_ReplacesAllFields_AndReturnsSuccess, Edit_WhenIsbnNotFound_ReturnsNotFoundAndChangesNothing, Edit_WithInvalidBook_ReturnsValidationFailure
Delete in: string isbn → out: Result Removes the <book> matching the given ISBN and saves. Returns a failed Result (NotFound) without writing anything if no book with that ISBN is found. Error Handling (Result & HTTP mapping)

Result.Ok / Result.Fail in: ResultError, string[] errors → out: Result Factories for the shared operation-outcome type. Success is derived from Error == None, so the two can never disagree. Errors lists every violation, not just the first. A null Book is reported as a caller error in the message, distinct from user-input violations. — ArgumentException if Fail is called with ResultError.None (programmer error, fails loudly at construction) exercised by every repository test asserting Success/Error
Result<T>.Ok / Fail in: T value / ResultError, string[] → out: Result<T> A Result that also carries a value on success, used by lookups (GetByIsbn). Value is default on failure. Result same as Result.Fail GetByIsbn_WhenIsbnNotFound_ReturnsNotFoundWithNoValue
ResultResponses.ToErrorResponse in: ApiController, Result → out: IHttpActionResult The one place a failed Result becomes HTTP: ValidationFailed→400, Conflict→409, NotFound→404; an unmapped kind fails loudly as 500. Body is ApiError: errors[] per violation plus the joined message for older clients. ApiError — Post_WithMultipleInvalidFields_ReturnsEachErrorSeparately, Post_DuplicateIsbn_Returns409, Put_UnknownIsbn_Returns404, Delete_UnknownIsbn_Returns404
GlobalExceptionHandler.Handle in: ExceptionHandlerContext → out: void (sets 500 response) Last-resort net for genuinely exceptional failures (registered in WebApiConfig): a schema-invalid data file gets an actionable safe message; anything else a generic one. Never leaks stack traces or exception detail. ApiError — Handle_SchemaViolation_Returns500WithSafeDataFileMessage, Handle_UnexpectedException_Returns500WithGenericMessage
