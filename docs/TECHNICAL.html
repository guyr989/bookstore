Prunes oldest-first beyond 100 kept versions. All mutations serialize on one process-wide gate so concurrent saves cannot claim the same number. — — Snapshot_CreatesNumberedVersionsStartingAtOne, Snapshot_CopiesTheCurrentFileContent, Snapshot_StampsTheTimeItWasTaken_NotTheDataFilesWriteTime, Snapshot_NumbersContinueFromTheHighestRemainingVersion
Restore in: int number → out: bool Copies the version back over the data file, returning false if it does not exist. Validates the snapshot against the XSD first (never rolls the live catalog back to a corrupted file) and stashes the current state before overwriting, so a restore is itself undoable — unless the data file is already gone, in which case restoring is the recovery path. BooksController

The HTTP surface over XmlBookRepository. Every action returns an IHttpActionResult; expected failures are never thrown — the repository's Result is routed through the single ToErrorResponse mapper (see Error Handling), so status codes live in one place. A schema-invalid data file is the one genuinely exceptional case and surfaces as a safe 500 via GlobalExceptionHandler.
